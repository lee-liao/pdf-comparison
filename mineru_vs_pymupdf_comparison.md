# MinerU vs PyMuPDF PDF提取能力对比报告

## 测试环境

- **PDF文件**: C19M7324501-1-1_mro.pdf
- **PDF总页数**: 23
- **页面尺寸**: 612 x 792 点 (Letter)

## 核心发现

### 1. 文本提取量

| 工具 | 提取字符数 | 相对比率 |
|------|-----------|---------|
| MinerU | 34,483 | 100% |
| PyMuPDF | 34,502 | 100.1% |

**结论**: 两者文本提取量基本相同，差异仅0.1%。

### 2. 结构化程度

#### MinerU - 高度结构化

第1页数据结构:
```
pdf_info[0]
└── para_blocks (1个)
    └── block type="table" (整页表格)
        └── blocks
            └── table_body
                └── lines
                    └── spans[0] type="table"
                        └── html="<table>...</table>" (1900字符)
```

**特点**:
- 整页表格作为一个语义单元
- 保留HTML结构
- 支持跨单元格、合并单元格
- 易于解析和理解

#### PyMuPDF - 碎片化

第1页数据结构:
```
get_text("dict")
└── blocks (43个)
    ├── text blocks (42个) - 单行或单词级别
    └── image blocks (1个)
```

**特点**:
- 表格被分解为42个独立的文本块
- 丢失表格结构
- 每个单元格成为独立的text block
- 难以重建原始表格布局

### 3. 表格处理对比

| 方面 | MinerU | PyMuPDF |
|------|--------|---------|
| 表格识别 | 1个table block | 1个TableFinder对象 |
| 单元格数量 | 13个(HTML解析) | 33个(非空) |
| 结构保留 | HTML格式 | 行列数组 |
| 数据完整性 | 高(含合并信息) | 中(纯文本) |

### 4. 内容质量对比

#### MinerU提取内容
```
CHINA SOUTHERN AIRLINES 中国南方航空公司 C19M7324501-1-1
A/C Type 机型 Registration No. 飞机注册号 ...
Job Card No.工卡号 C19M7324501-1-1 Rev. Date 版本日期 2025-11-25 R1
...
```
- 连续的语义单元
- 表格内容完整

#### PyMuPDF提取内容
```
树形签署
A/C Type
机型
C919
...
Job Card No. 工卡号
...
```
- 碎片化的行
- 表格单元格分散

### 5. 关键词检测

| 关键词 | MinerU | PyMuPDF |
|--------|--------|---------|
| 工卡号 | 10 | 23 |
| 机 | 68 | 82 |
| Aircraft | 24 | 24 |
| WARNING | 12 | 12 |
| CAUTION | 22 | 22 |
| NOTE | 10 | 9 |

**分析**: PyMuPDF检测到更多的"工卡号"和"机"，这是因为PyMuPDF保留了更多碎片化的文本。

## 差异影响分析

### 对于PDF差异比较

**问题**: 为什么PyMuPDF不适合PDF差异比较？

1. **语义单元丢失**
   - MinerU: 表格作为一个整体，删除整行=删除一个语义单元
   - PyMuPDF: 删除整行=删除43个碎片，差异检测混乱

2. **顺序不一致**
   - MinerU: 按语义结构组织（table → lines → spans）
   - PyMuPDF: 按物理位置组织（碎片化的blocks）

3. **比较颗粒度**
   - MinerU: 适合段落/表格级别的比较
   - PyMuPDF: 过于细碎，产生大量"删除/新增"噪音

### 实际例子

假设PDF表格中删除了一行：

**使用MinerU**:
```
- [删除] 整个表格行
```

**使用PyMuPDF**:
```
- [删除] text block 1
- [删除] text block 2
- [删除] text block 3
... (共43个操作)
```

## 技术对比表

| 维度 | MinerU | PyMuPDF |
|------|--------|---------|
| **提取速度** | 较慢（API调用） | 快速（本地处理） |
| **文本量** | 34,483字符 | 34,502字符 |
| **结构化** | 高（HTML/JSON） | 低（纯文本） |
| **表格支持** | 完整（HTML结构） | 部分（行列数据） |
| **数学公式** | 支持（LaTeX） | 不支持 |
| **语义理解** | 强（保留文档结构） | 弱（碎片化） |
| **差异检测** | 优秀 | 较差（噪音多） |
| **API依赖** | 是（在线服务） | 否（本地库） |
| **文件大小** | JSON较大 | 文本较小 |

## 建议

### 使用MinerU的场景
- PDF差异比较
- 结构化内容提取
- 表格数据解析
- 需要保留原始布局
- 精确的内容定位

### 使用PyMuPDF的场景
- 快速文本预览
- 简单的文本搜索
- 页面/图片提取
- 本地离线处理
- 不需要结构化的场合

## 结论

对于PDF差异比较应用，**MinerU是必需的**。PyMuPDF虽然能提取相同数量的文本，但碎片化的数据结构会导致差异检测质量大幅下降，产生大量噪音。
